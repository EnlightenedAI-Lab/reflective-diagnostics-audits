What this site is

This site shows selected audit runs from Reflective Diagnostics.

The goal is simple: not just to ask whether a model got the answer right, but to look at how it answered.

Some models are wrong but sound very confident. Some get the answer right but lose structure. Some stay stable across reruns. Others drift into different answers. This site highlights those differences.

What the system looks at

Main questions

  • Did the model stay consistent?
  • Did it support its claims?
  • Did revision improve the answer?
  • Did confidence match the evidence?
  • Did it hold up under pressure?
  • Did it fail carefully or hallucinate confidently?

Main parts

  • Probe — runs prompts across models
  • Reasoning — checks support and structure
  • Reflection — compares first and revised answers
  • Uncertainty — checks rerun stability
  • Epistemic Discipline — checks confidence versus support
  • Integrity — checks task and constraint grounding
  • Logprobs — checks generation instability
  • Elasticity — tests hold, snap, or recovery under pressure
  • Battery Runner — runs benchmark batches across models

Benchmark families

The current benchmark families include:

Mathematics

Tests symbolic and multi-step computation.

Bioinformatics

Tests long-sequence positional reasoning.

Cryptography

Tests whether models admit limits or hallucinate impossible exact answers.

Graph Theory

Tests relational working memory and shortest-path reasoning.
