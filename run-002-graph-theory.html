Hypothesis

Large context windows should not be confused with strong relational reasoning. A model may retain large amounts of text while still failing when required to compute exact structure over dense graph relationships. This run tests whether frontier models can reliably solve shortest-path problems under adversarial graph complexity while also preserving output structure and calibrated confidence.

Run profile

This batch run evaluated five frontier models on twenty graph-theory adversarial items, for a total of one hundred scored records. The overall result was mixed rather than catastrophic: 47.0% exact-answer accuracy, 90.0% parse rate, and 50.0% structural decay across the batch. The run also recorded 40 confident mistakes and an overall arrogance score of 72.55.

Core finding

This run does not support a total-collapse narrative. It shows something more useful: strong separation between relational accuracy, format retention, and epistemic posture.

Some models preserved format while remaining badly overconfident when wrong. Others lost structural discipline under load. One lane stayed cautious but suffered severe parse weakness. The important result is not only who scored highest. The more important result is that this batch exposed distinct failure styles: format-stable overconfidence, structurally degraded reasoning, parse fragility, and cautious partial success.

Why it matters

A standard benchmark would mostly reduce this run to answer accuracy. This case shows why that is not enough. Two models can post similar scores while failing in very different ways. One can remain perfectly formatted yet hallucinate confidently. Another can remain cautious but fail to complete the task reliably. Another can solve a subset correctly while shedding structure under pressure.

Bottom line: adversarial relational tasks remain a meaningful discriminator across frontier models, especially when correctness, parse stability, structural discipline, and confidence behavior are separated instead of collapsed into one score.
