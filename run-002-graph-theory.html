What was tested

This run tested five frontier models on graph problems.

Each task asked for the exact shortest path distance between two nodes in a dense graph.

The point was not only to see who got the answer right. The point was also to see:

  • who stayed structured
  • who became overconfident
  • who broke under the task

Why this test matters

Many models advertise very large context windows.

But holding a lot of text is not the same as doing strong relational reasoning. A model may keep a large prompt in memory and still fail when it has to track many connections exactly.

This test was built to check that difference.

Run setup

  • Module: L15 Battery Runner
  • Benchmark family: Graph Theory Oracle
  • Models: GPT-5.2, Claude Opus 4.6, Gemini 2.5 Flash, DeepSeek Chat, Grok-3
  • Batch size: 20 graph items per model
  • Total records: 100

Overall result

This was a difficult run.

Across the full batch, the result was:

  • 47.0% accuracy
  • 90.0% parse rate
  • 50.0% structural decay
  • 40 confident mistakes

So this was not a total collapse, but it was a strong stress test. It clearly separated different failure styles.

Main finding

This run showed that three things can come apart:

  • answer accuracy
  • output structure
  • confidence behavior

A model can keep perfect format and still be confidently wrong.

A model can stay cautious but fail to complete reliably.

A model can solve some tasks correctly but lose structure under load.

That is why this kind of audit is useful.

Why this is better than a simple benchmark

A normal benchmark would mostly reduce this run to a score.

This audit keeps more information:

  • who was correct
  • who stayed stable
  • who broke format
  • who became overconfident
  • who failed cautiously
Bottom line: dense graph reasoning remains a useful stress test for frontier models, especially when correctness, structure, and confidence are measured separately.
